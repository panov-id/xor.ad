# relay — усиление и enterprise-роадмап

Что уже сделано и что дальше, чтобы довести relay до enterprise-уровня. Дополняет
`SPEC_{EN,RU}.md` (архитектура) и `RELEASE_{EN,RU}.md` (доставка).

## Сделано

- **Мультиарх-образы.** CI собирает `linux/amd64` + `linux/arm64` (боксы бывают ARM —
  Hetzner CAX, Oracle A1 — и x86 — GCP). Одноарховый образ на ARM не запустится.
- **Supply chain в CI:** SBOM + SLSA-provenance на каждый образ; **cosign**
  keyless-подпись; **Trivy**-скан уязвимостей (HIGH/CRITICAL, отчёт). Зависимости
  зафиксированы через **`deno.lock`**.
- **Наблюдаемость (нода):** структурные **JSON-логи** (level/msg/поля, node, env,
  request id) → готовы под Loki/ELK; **`GET /metrics`** Prometheus-счётчики
  (`relay_requests_total`, `relay_waitlist_total`, `relay_mail_total`); заголовок
  `x-request-id` в каждом ответе.
- **Доступ/безопасность (см. SPEC §9):** firewall default-deny, SSH только по ключу
  с хардненингом, TLS через DNS-01 с закрытыми портами, гейт прод = опубликованный
  релиз + `--confirm-prod`.
- **Доставка (см. RELEASE):** build-once-promote, неизменяемые version-теги.
- **Тесты + CI:** unit + integration на каждое изменение `relay/**`.

## Роадмап (по приоритету)

### P1 — ближайшее, высокая ценность
- **Централизованная наблюдаемость.** Слать логи ноды + `/metrics` в **Grafana +
  Loki + Prometheus** (self-host на маленьком боксе или Grafana Cloud free).
  **Алерты:** нода упала, истечение сертификата, error-rate, сбой почты. Dozzle
  на каждом боксе остаётся для быстрых логов.
- **Гигиена секретов.** Заменить широкий PAT и аккаунт-ключ Bunny на боксах на
  **минимальные права**: токен только `read:packages`, узкий DNS-ключ Bunny.
  Перенести `secrets.env` в **SOPS/age** (или secret-manager); политика ротации.
- **Защита публичного `/waitlist`.** **Bunny Shield** rate-limit + **Turnstile**
  captcha до прода; throttling по IP; ужесточение валидации.
- **Health-монитор гео-пула (прод).** Bunny DNS health check на записи `api.<лицо>`,
  чтобы мёртвая нода автоматически выпадала из ротации.
- **PII / GDPR.** Email — персональные данные: политика хранения, эндпоинт
  **удаления** (запрос субъекта), описанные потоки данных, DPA с вендорами,
  подтверждение шифрования at-rest.

### P2 — глубже в enterprise
- **IaC.** Terraform (`hcloud` + `bunnynet`) для боксов и DNS с plan/apply/state —
  воспроизводимая инфраструктура вместо (или генерируемая) визарда.
- **Progressive delivery.** Canary / по одной ноде с **авто-откатом** по health;
  DORA-метрики.
- **Аудируемый гейт аппрува.** Self-hosted runner (на whitelisted admin-хосте) +
  GitHub Environments, чтобы промоушен на прод был ревьюируемым логируемым шагом.
- **Бэкап + DR.** Экспорт waitlist + **проверенное восстановление**; DR-runbook.
- **Глубина тестов.** Покрытие роутов/транспортов/ошибок, **нагрузка/перф**
  (будущий чат-WS), **chaos** (убить ноду), тесты обхода авторизации.

### P3 — по мере роста платформы
- **Трейсинг.** OpenTelemetry сквозь BFF → core (вместе с приложением/чатом).
- **SSH в масштабе.** SSH-сертификаты через CA / just-in-time доступ / бастион.
- **mTLS** между компонентами; **проверка cosign-подписи** образа на боксе перед
  запуском (admission).

## Заметки

- Образы relay **не содержат секретов** (конфиг приходит через env в рантайме),
  поэтому сделать ghcr-пакеты публичными — безопасная альтернатива box-login.
- `--confirm-prod` + опубликованный релиз — текущий гейт прода; аппрув через GitHub
  Environment — добавка уровня P2 (нужен self-hosted runner, т.к. боксы в
  IP-whitelist).
