# Панель — админ/модераторская панель для xor.ad

## Назначение

Внутренний интерфейс для команды: видимость и управление жалобами, банами и квотами пользователей на обеих витринах (sosed.place, neighbro.place). Это операционная поверхность общего бэкенда, отдельная от пользовательских фронтендов.

## Функции

- Просмотр жалоб на сообщения и пользователей.
- Бан пользователя (по UID).
- Просмотр и ручная корректировка квоты публикаций пользователя.
- Просмотр обращений в поддержку (см. шаг "Поддержка" в README) — тикеты из Supabase с уведомлением по email/webhook.
- Управление каталогом стикеров (за пределами альфы, см. README): загрузка картинок через Storage/imgproxy, редактирование фиксированного каталога, доступного пользователям для покупки за баланс в приложении.

## Логика

- Панель работает поверх общего бэкенда (Supabase) — те же данные, что видят фронтенды, но с правами модератора/админа.
- Один бэкенд — одна панель для обеих витрин, с фильтром по лицу (sosed.place/neighbro.place) там, где это нужно (например, для разной политики по ЛГБТ-контенту — см. раздел «Модерация» в README).

## Открытые вопросы

- Технологии/фреймворк самой панели не определены.
- Ролевая модель доступа (кто именно может банить/менять квоты) не определена.
- Способ аутентификации команды в панели (отдельный логин, Supabase Auth с ролями) не определён.
