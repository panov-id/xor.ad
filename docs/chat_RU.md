# Чат — спецификация

Единый документ по функционалу чата. Сводит воедино требования из `app-prototype-spec_RU.md`, `app-ui-notes_RU.md`, `backlog_RU.md`, `roadmap_RU.md`. Статус: основа (реализация — отдельными шагами).

## 1. Назначение

Чат — приватный **эфемерный** разговор двух людей, который открывается **только после взаимного лайка** (мэтча) в ленте города. Это не мессенджер: чат живёт ограниченное время и исчезает вместе с контекстом. Задача — «общий момент» здесь и сейчас, а не переписка навсегда.

Принципы:
- **Эфемерность** — у чата есть срок жизни; по истечении он угасает и пропадает.
- **Контекст мэтча** — видно, из-за чего люди состыковались (лайкнутые фразы).
- **Лёгкость входа** — ice-breaker важнее «функций»: короткие сообщения, совместные игры без правил.

## 2. Модель мэтча

Путь от ленты до открытого чата:

1. **Лайк** — тап по кнопке-логотипу на сообщении в ленте (реакция на фразу).
2. **Взаимный лайк** — если второй человек лайкнул ваше сообщение → появляется запрос.
3. **`Requests`** — раздел «likes you back →»: взаимные лайки, чат **ещё не открыт**.
4. **Открытый чат** — подтверждённый мэтч; чат доступен в списке `Chats`, запускается таймер жизни.

Системное сообщение при открытии: `you both liked this — chat is open`.

## 3. Список чатов

- Табы: **`Chats N`** / **`Requests N`** (брутал-блоки, активный — акцентная заливка).
- **Тред** (`.thread`): аватар-буква, имя, время последней активности, **превью последнего сообщения** (отдельной строкой), **таймер** `chat open · Nh Nm` (акцентом, отдельной строкой), шеврон `›` как подсказка кликабельности.
- **`Requests`** — «likes you back →» (взаимные лайки, чат ещё не открыт); действие — открыть чат.
- Клик по треду → открывает разговор.

## 4. Разговор (conversation)

- Шапка: кнопка «назад», имя + таймер `chat open · Nh Nm` (акцентом), кнопка 🎲 **«предложить игру»**.
- **Лайкнутые фразы (контекст мэтча):** в начале разговора блок `Liked, in order` — пронумерованный список фраз в **порядке лайка**, с меткой `You liked` / `They liked` и цитатой. Показывает, из-за чего вы состыковались.
- **Пузыри:**
  - `them` — слева, поверхность `--ink-2`, тень-сдвиг;
  - `me` — справа, акцентная заливка `--gold` + `--gold-ink`;
  - `sys` — по центру, mono (системные события: открытие чата, предложение игры);
  - у каждого пузыря время (`HH:MM`).
- **Композер** (`.composer`): поле `Write a message…` (`maxlength=240`) + кнопка отправки (стрелка, акцентная заливка). Submit добавляет пузырь `me` и скроллит вниз.

## 5. Эфемерность

- У чата есть **TTL** (срок жизни), отображается таймером `chat open · Nh Nm`.
- По мере приближения к истечению — визуальное **угасание**; по истечении чат **пропадает** вместе со всем содержимым (сообщения, лайкнутые фразы, игровая доска).
- Срок хранится на сервере (привязка к личности-браузеру), не только в localStorage.
- Открытые вопросы: продлевается ли TTL при активности; единый ли срок для всех чатов; поведение ровно в момент истечения (мгновенно скрыть / показать «истёк»).

## 6. Совместные игры без правил

Внутри чата — общая визуальная доска для двоих: **домино, шашки, шахматы**. Фишка: **никаких зашитых правил** — движок только рисует доску и даёт свободно двигать/ставить фигуры; правила игроки придумывают и соблюдают сами. Это ice-breaker, а не соревнование.

- Запуск/переключение **по запросу**: кнопка 🎲 «предложить игру» → выбор доски → другому уходит запрос → принял → доска открывается у обоих. Смена игры — тем же запросом.
- Нет валидации ходов, счёта, победителя — только состояние доски + перетаскивание.
- Двигать фигуры могут оба (правил нет).
- Доска живёт в рамках чата и **исчезает вместе с ним** (эфемерность).
- Синхронизация — в реальном времени (см. §7).

## 7. Realtime

- Обмен сообщениями, состояние игровой доски, запросы на игру — через **Supabase Realtime** (вебсокеты) поверх **api-прокси** (same-origin, через gateway).
- Новые сообщения приходят без перезагрузки; лента города обновляется отдельно (`Refresh` / `Auto` ~6с).

## 8. Модель данных (требования к схеме)

Уровень требований — без миграции (SQL и RLS — отдельным шагом, см. `roadmap` п.50–51).

- **`likes`** — кто кого/какое сообщение лайкнул; из взаимных лайков выводятся `Requests`.
- **`chats`** — открытый чат между двумя личностями; `created_at`, `expires_at` (TTL), состояние.
- **`messages`** — сообщения чата: автор, текст (≤240), время; тип `me`/`them` вычисляется по зрителю; `sys` — системные.
- **`game_sessions`** — активная доска в чате: тип (домино/шашки/шахматы), состояние фигур; живёт в рамках чата.

Требования:
- **RLS**: доступ к чату/сообщениям/игре — только двум участникам.
- **Эфемерность**: TTL на уровне БД (фоновая очистка/фильтр по `expires_at`).
- Realtime включён для `messages` и `game_sessions`.

## 9. UI-состояния и брейкпоинты

- **`≥900px`** — трёхколоночный рабочий стол: **[Лента] | [Открытые чаты] | [Активный чат]** (лента `flex:1`, чаты `300px`, активный чат `400px`). Колонки сворачиваются в вертикальные рейлы. До выбора чата активная колонка — пустое состояние `Pick a chat`.
- **`≤899px`** — одна колонка, нижняя навигация (`Feed` / `Chats` / `Say` / `Me`); разговор — полноэкранный оверлей (`position:fixed`), «назад» → список.
- **`≤560px`** — сжатый хедер.

## 10. Доступность / качество

- `:focus-visible` — акцентный контур; `prefers-reduced-motion` — отключает угасание/пульсацию.
- `overflow = 0` по горизонтали во всех состояниях (список / разговор / игра), базовый экран — iPhone 12 mini (375px).

## 11. Логотип: домик и текст (единое поведение — лендинг и приложение)

Логотип состоит из двух кликабельных частей с **разными** действиями. Правило одинаково на лендингах (sosed.place / neighbro.place) и в приложении.

- **Знак-домик** — меняет тему (как сейчас). На лендинге это цикл акцентного цвета (кнопка `#logoBtn`); свет/тьма — отдельная кнопка ☀/🌙. Поведение домика не меняется.
- **Текст названия** (`SOSED` / `NEIGHBRO`) — переход **«на главную»**, где «главная» зависит от авторизации:
  - **зареган** (активная Supabase auth-сессия) → **окно чата** приложения;
  - **не зареган / вышел** → **лендинг**.
- **Определение «зареган»** = активная Supabase auth-сессия. Пока вход не реализован, сессии нет → текст-лого всегда ведёт на лендинг.
- Доступность: текст-лого — семантическая ссылка/кнопка с `aria-label`; домик и текст различимы по фокусу.
