# Дизайн приложения — аудит + чеклист новой концепции

Статус: заметки/чеклист. Код — отдельным шагом. Относится к самому приложению (не лендингу); эстетика тянется из лендинга neighbro (тёмный/золото) + добавляем брутализм и бетон.

## Аудит текущего (лендинг neighbro)

Что работает и переносим в апп:
- Тёмная база + золото, шрифты Unbounded (дисплей) / Golos Text (текст) / JetBrains Mono (лейблы).
- Крупная дерзкая типографика, mono-лейблы, чёткая иерархия.
- Лого-знак (дом + хвостик чата) — сильный, годится как иконка-кнопка.

Что не нравится / переделать:
- **Превью ленты** — слишком аккуратный «список чата», скучно и не передаёт живость/эфемерность. Переделать в «разбросанную» ленту (см. ниже).
- Мало «характера»: сейчас элегантно, но не брутально. Добавить **бетон и брутализм** — тяжёлые рамки, сырые бетонно-серые поверхности рядом с золотом, обнажённая сетка, блочные сдвиги.

## Общая эстетика: брутализм + бетон

- [ ] Добавить бетонную палитру рядом с тёмным/золотом: серый бетон (`#6b6b66`/`#3a3a37`/`#1a1a18`), возможно лёгкая зернистая текстура (SVG-noise) на поверхностях.
- [ ] Жёсткие толстые рамки (3–5px), острые углы, жёсткие тени-сдвиги без блюра.
- [ ] Обнажённая структура: видимая сетка/разделители, mono-лейблы как «технические подписи».
- [ ] Золото — точечный акцент (кнопки/лого/активное), не заливка.

## Широкий экран: сплит фид + чаты

- [ ] Двухколоночный лейаут: **слева фид** (листается), **справа чаты**.
- [ ] Справа — **табы** переключения (напр. «Лента города» / «Чаты»), сделать удобно и по трендам (крупные, тактильные, с явным активным состоянием; можно в брутал-стиле — блочные табы с жёсткой рамкой).
- [ ] Обе колонки скроллятся независимо.
- [ ] Порог ширины, ниже которого сплит схлопывается в одноколоночный (мобильный лейаут).

## Лента (фид) — «разбросанные» сообщения

- [ ] Сообщения **разбросаны** по площади (не ровный список): разные позиции/размеры, коллаж/мэйсонри, лёгкий наклон/сдвиг — «живая доска», а не таблица.
- [ ] У каждого сообщения **своя механика взаимодействия**; основное действие — **кнопка-логотип** (знак бренда = кнопка лайка/реакции на сообщении). Тап по лого = лайк/реакция (→ возможный мэтч).
- [ ] Листать ленту прокруткой.
- [ ] Тап по сообщению → **открывается на весь экран**; листание **свайпом влево/вправо** (пальцем «вжух») между сообщениями.
- [ ] Полноэкранный режим: крупный текст, мета (дистанция/таймер/сколько нас), действие через лого-кнопку.

## Вертикальный экран (мобайл)

- [ ] **Нижняя навигация** (bottom nav) — как везде: 3–5 кнопок (напр. Лента / Чаты / Добавить / Профиль/Настройки). Брутал-стиль: блочные, с жёсткой верхней рамкой.
- [ ] Сплит фид/чаты недоступен — переключение через нижние табы.
- [ ] Кнопка «добавить сообщение» — заметная (центральная в нижней навигации или FAB).

## Открытые вопросы

- Точный набор кнопок нижней навигации.
- Что именно значит «механика» у каждого сообщения помимо лайка (реакции? таймер-визуал? свайп-действия?).
- Насколько сильный «разброс» сообщений, чтобы оставалось читаемо и доступно (a11y, отсутствие горизонтального оверфлоу).
- Табы справа: только «Чаты» или ещё под-разделы (запросы на чат, активные, история мэтча).
- Бетонная текстура: SVG-noise inline или CSS-паттерн (без внешних ассетов, чтобы работало в PWA/CSP).
