# relay — регламент релизов и промоушена

**Принцип: собираем релиз один раз, катим один и тот же образ dev → staging → prod.**
Что протестили — то и в проде, байт-в-байт. Никакой пересборки на каждый env.

## Поток

```
локально ──тест── ветка dev ──CI──▶ dev env (sha-образ, авто)
                                       │  смоук + ручная протыкка
                                       ▼
                            merge dev → main
                                       │  тег vX.Y.Z (+ GitHub Release notes)
                                       ▼
                     CI собирает релизный образ :vX.Y.Z (раз) ──▶ ghcr
                                       │
                                       ▼  катим ТОТ ЖЕ :vX.Y.Z
                                     staging env
                                       │  смоук + ручная протыкка
                                       ▼  ручное подтверждение (prod-гейт)
                                     prod env   ← тот же :vX.Y.Z, без пересборки
```

1. **Разработка локально** — стенд `local/` (`docker compose up`).
2. **Тесты локально** — `cd node && deno test` + `bash test/integration.sh`.
3. **Push в `dev`** → CI: тесты, сборка `edge-node:<sha>` / `edge-caddy:<sha>`,
   авто-деплой на **dev** (sha-образ — быстрая итерация, не формальный релиз).
4. **Проверка dev** — пост-деплой смоук (авто) + ручная протыкка.
5. **Merge `dev` → `main`** когда всё устраивает.
6. **Режем релиз** — ставим **ручной semver-тег `vX.Y.Z`** (или GitHub Release).
   CI собирает `edge-node:vX.Y.Z` / `edge-caddy:vX.Y.Z` **один раз** → ghcr.
7. **Катим `:vX.Y.Z` на staging.**
8. **Проверка staging** — пост-деплой смоук + ручная протыкка.
9. **Промоут того же `:vX.Y.Z` на prod** — визард требует `--confirm-prod` и
   чтобы `vX.Y.Z` был **опубликованным GitHub Release** (публикация = approve).
   Без пересборки.

## Правила

- **Иммутабельные версии.** Релизы пинятся по `:vX.Y.Z`, никогда `:latest`.
  Инвентарь каждого env пинит свой тег.
- **Пост-деплой смоук на каждом env.** После деплоя: дёрнуть `/health`, затем
  синтетический `POST /waitlist` на тест-адрес (dev/local → в Mailpit).
- **Prod-гейт = опубликованный релиз.** Prod-деплой требует `--confirm-prod` И
  чтобы `image_tag` env был **опубликованным GitHub Release** `vX.Y.Z` (визард
  проверяет через API). Публикация релиза = approve — без инфры и раннера;
  непротестированный/недорелизнутый билд просто не доедет до prod.
- **Откат = задеплоить предыдущий `:vX.Y.Z`** на затронутом env (одной командой).
- **Release notes** на каждом GitHub Release.

## Env ↔ артефакт

| Env | Артефакт | Триггер | Гейт |
|-----|----------|---------|------|
| dev | `:<sha>` (ветка dev) | push в `dev` | авто |
| staging | `:vX.Y.Z` (релиз) | деплой тега | после зелёного dev |
| prod | **тот же** `:vX.Y.Z` | промоут тега | ручной approve |
