# relay — hardening & enterprise roadmap

What's in place and what's next to move relay toward enterprise-grade. Companion
to `SPEC_{EN,RU}.md` (architecture) and `RELEASE_{EN,RU}.md` (delivery).

## Done

- **Multi-arch images.** CI builds `linux/amd64` + `linux/arm64` (boxes are ARM —
  Hetzner CAX, Oracle A1 — and x86 — GCP). Single-arch would not run on ARM.
- **Supply chain in CI:** SBOM attestation + SLSA provenance on each image;
  **cosign** keyless signature; **Trivy** vulnerability scan (HIGH/CRITICAL,
  report). Deps pinned via **`deno.lock`**.
- **Observability (node):** structured **JSON logs** (level/msg/fields, node, env,
  request id) → ready for Loki/ELK; **`GET /metrics`** Prometheus counters
  (`relay_requests_total`, `relay_waitlist_total`, `relay_mail_total`); an
  `x-request-id` on every response.
- **Access/security (see SPEC §9):** default-deny firewall, key-only hardened SSH,
  DNS-01 TLS with ports locked, prod gate = published-release + `--confirm-prod`.
- **Delivery (see RELEASE):** build-once-promote, immutable version tags.
- **Tests + CI:** unit + integration, gated on every `relay/**` change.

## Roadmap (prioritized)

### P1 — near-term, high value
- **Centralized observability.** Ship node logs + `/metrics` to **Grafana + Loki +
  Prometheus** (self-host on a small box or Grafana Cloud free). **Alerting:** node
  down, cert-expiry, error-rate, mail-failure. Per-box Dozzle stays for quick logs.
- **Secret hygiene.** Replace the broad PAT + account-wide Bunny key on boxes with
  **least-privilege**: a `read:packages`-only token, a scoped Bunny DNS key. Move
  `secrets.env` to **SOPS/age** (or a secret manager); rotation policy.
- **Public `/waitlist` protection.** **Bunny Shield** rate-limit + **Turnstile**
  captcha before prod; per-IP throttling; input hardening.
- **Geo-pool health monitor (prod).** Bunny DNS health check on the `api.<face>`
  record so a dead node drops out of rotation automatically.
- **PII / GDPR.** Emails are personal data: retention policy, a **deletion**
  endpoint (data-subject request), documented data flows, vendor DPAs, at-rest
  encryption confirmation.

### P2 — deeper enterprise
- **IaC.** Terraform (`hcloud` + `bunnynet` providers) for boxes + DNS with
  plan/apply/state — reproducible infra instead of (or generated by) the wizard.
- **Progressive delivery.** Canary / one-node-at-a-time rollout with health-gated
  **auto-rollback**; DORA metrics.
- **Auditable approval gate.** Self-hosted runner (on the whitelisted admin host)
  + GitHub Environments so prod promotion is a reviewed, logged Actions step.
- **Backup + DR.** Waitlist export + **tested restore**; documented DR runbook.
- **Test depth.** Route/transport/error-path coverage, **load/perf** (the future
  chat WS), **chaos** (kill a node), auth-bypass tests.

### P3 — as the platform grows
- **Tracing.** OpenTelemetry across BFF → core (with the app/chat).
- **SSH at scale.** CA-signed SSH certs / just-in-time access / a bastion.
- **mTLS** between components; **image cosign verification** enforced on the box
  before run (admission).

## Notes

- The relay images carry **no secrets** (config comes via env at runtime), so
  making the ghcr packages public is a safe alternative to the box login.
- `--confirm-prod` + published-release is the current prod gate; the GitHub
  Environment approval is a P2 add-on (needs a self-hosted runner because boxes
  are IP-whitelisted).
